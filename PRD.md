# 마음의 편지함: 제품 요구사항 정의서 (PRD)

## 1. 사용자 및 익명성
### 1.1. 익명 닉네임
- **닉네임 할당**: 사용자가 처음 접속하면 고유한 익명 닉네임이 자동으로 할당됩니다.
  - 예: `"당신의 이름은 "버터의추종자"입니다."`
- **닉네임 형식**: `{트릭컬 사도}의{호칭}` 조합으로 생성됩니다.
  - 예: `캬롯의지휘관`, `시스트의대리인`
- **닉네임 변경**: 사용자는 스스로 닉네임을 변경할 수 없습니다.

### 1.2. 사용자 식별
- **식별 방식**: IP 주소가 아닌, 브라우저의 `LocalStorage` 또는 `Cookie`에 저장된 고유 식별자(**UUID**)를 기준으로 사용자를 식별합니다.
  - **기대 효과**: 유동 IP나 공용 IP 환경에서도 사용자의 연속적인 활동을 보장합니다.

### 1.3. 관리자 권한
- 관리자는 서비스 운영 및 제재 목적으로 특정 사용자의 닉네임을 강제 변경하거나 활동을 제한할 수 있습니다.

---

## 2. 편지함 (게시판)
### 2.1. 주차별 운영
- **운영 주기**: 매주 월요일 00시를 기준으로 새로운 '주차(시즌)'가 시작됩니다.
  - 예: `마음의 편지함 1주차`, `마음의 편지함 2주차`
- **아카이빙**: 이전 주차의 모든 글은 **읽기 전용** 상태로 보관(아카이브)됩니다.

### 2.2. 게시글 목록
- **표시 정보**: 글 번호, 제목, 작성자(익명), 조회수, 추천 수
- **정렬**: 최신순으로 내림차순 정렬됩니다.

---

## 3. 게시글 기능
### 3.1. 글 열람
- **표시 정보**: 추천/비추천 수, 관리자 답글, 조회수, 작성일(날짜와 시간)
- **사용자 상호작용**:
  - 모든 사용자는 글에 대해 **추천** 또는 **비추천**을 할 수 있습니다.
- **개념글**: 아래 기준을 충족하는 게시글은 '개념글'로 자동 선정됩니다.
  - `비추천 수 대비 추천 수가 3배 이상`
  - `총 추천 수가 5개 이상`
  - 유지보수에 따라 기준이 달라질 수 있음

### 3.2. 글 작성
- **작성자**: 글 작성 시, 본인의 익명 닉네임이 자동으로 표시됩니다.
- **제목**: 일반 텍스트만 입력 가능합니다.
- **내용**: **블록 기반 에디터(Block-based Editor)**를 사용합니다.
  - **구현**: `Editor.js`, `Quill.js` 등 검증된 오픈소스 라이브러리 사용을 권장합니다.
  - **블록 종류**:
    - 텍스트 (Tt)
    - 미디어 (사진/비디오)
    - 임베드 (유튜브 등)
  - **블록 제어**: 각 블록의 순서를 변경하거나 삭제할 수 있는 UI(핸들러, 삭제 버튼)를 제공합니다.
  - **편의 기능**: 작성 중인 내용이 자동으로 임시 저장되어, 페이지 이탈 시에도 복구할 수 있도록 합니다.
- **유효성 검사**: 제목이 비어있거나 내용 블록이 하나도 없는 경우, 글을 등록할 수 없으며 사용자에게 알림(`Alert`)을 표시합니다.

### 3.3. 글 수정
- **권한**: 작성자 본인만 글을 수정할 수 있습니다. (브라우저의 고유 식별자로 권한 확인)

### 3.4. 글 삭제
- **권한**: 작성자 본인 또는 관리자만 글을 삭제할 수 있습니다.
- **삭제 방식**: **논리적 삭제(Soft Delete)** 방식을 사용합니다.
  - 데이터베이스에서 실제 데이터를 삭제하는 대신, '삭제됨' 상태로 변경합니다.
  - 사용자에게는 `"삭제된 게시물입니다."`와 같이 표시됩니다.

---

## 4. 관리자 기능
### 4.1. 관리자 답변
- 관리자는 모든 게시글에 공개적인 답변(댓글)을 작성할 수 있습니다.

### 4.2. 사용자 관리
- **활동 조회**: 특정 닉네임의 활동 내역(작성 글, 댓글, 추천/비추천 내역)을 조회할 수 있는 관리자 전용 페이지를 제공합니다.
- **상호작용 추적**: 특정 글에 추천 또는 비추천을 한 사용자들의 닉네임 목록을 열람할 수 있습니다.
- **닉네임 강제 변경**: 특정 닉네임이 부적절한 경우 강제로 변경할 수 있습니다.
- **글 강제 삭제**: 특정 글이 부적절한 경우 강제로 삭제할 수 있습니다.
- **댓글 강제 삭제**: 특정 댓글이 부적절한 경우 강제로 삭제할 수 있습니다.

### 4.3. 추천/비추천 관리
- **추천/비추천 취소**: 사용자가 추천/비추천한 글에 대해 추천/비추천을 취소할 수 있습니다.
- **추천/비추천 내역 조회**: 특정 글에 추천/비추천한 사용자들의 닉네임 목록을 열람할 수 있습니다.
- **추천/비추천 내역 삭제**: 특정 글에 추천/비추천한 사용자들의 닉네임 목록을 삭제할 수 있습니다.
